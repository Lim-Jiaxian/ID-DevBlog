- Week 7 -

In week 7, the topics of DOM was taught. For instance, when a browser loads up a webpage or site, it creates a model of that webpage titled DOM Tree. DOM also knowned as Document Object Model, is a cross-platform interface that houses html documents and each node is an object that represents a certain part of a document. It is somewhat similar to a file hierarchy, where different objects are stored in different nodes which allows accesibility to change the style, structure and the content of the document. To access the elements we could use getElementById, querySelector or even getting the class / tag name. To modify the element, the process breaks down in several parts. First off, when the code selects an element to modify, the interpreter will make use of the DOM Tree to search for it's contents. Followed by identifying the class ID attribute and the value. Once found, you can start making changes / edits to that individual note that has either parent or child.

For the activity this week, a JavaScript program was used to loop different elements in the class and updated it's attributes. For e.g. class "li.hot" was changed to "cool". However, since i was only required to change the 3rd element that has the class attribute of hot to cool, the loop was used to search for the 3rd element by using getElementByClassName. Apart from that, I'm also tasked with creating a JavaScript file that allows lower case letters entered to a textbox automatically changed into upper case. To achieved this, getElementById was used to obtain the element, followed by using addEventListener to get the target value before setting it to uppercase.

In essence, AJAX let you ask for information from a web server that javascript processses and returns.
